Important FAQ

What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page.

How do you select an element from the DOM?

I can select an element from the DOM, Using getElementsByTagName(), Using getElementById(), Using getElementsByClassName(), Using querySelector() and Using querySelectorAll()

What is event delegation in the context of the DOM, and why is it useful?

Event delegation refers to the process of using event propagation (bubbling) to handle events at a higher level in the DOM than the element on which the event originated. It allows us to attach a single event listener for elements that exist now or in the future. Event delegation is a design pattern in JavaScript used to efficiently manage and handle events on multiple child elements by attaching a single event listener to a common ancestor element.

How do you manipulate on element's attributes and styles using the DOM?

I can use getAttribute(), setAttribute() and removeAttribute() for working with an element’s attributes, or access them directly as properties. I can modify an element’s inline styles using the style property, or manipulate CSS classes using the classList API for cleaner and more maintainable code.
